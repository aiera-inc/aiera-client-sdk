DeepPartial

DeepPartial<T>: {[ P in keyof T]: DeepPartial<T[P]> }
  • Defined in src/types/index.ts:62

Partial only goes one level deep, this makes all the nested types partials as well.

The main use case for this is working with GQL results.

If you have a utility function that you want to work on Company but you only queried some of the fields on Company (and it's child fields) which is almost always the case, then you'd get an error when passing the company field through even if you handle null/undefined properly. This allows you to specify the input type as DeepPartial so that it works and forces you to handle null/undefined.

If the function can't handle null/undefined, specify the full expected shape in your input type.
